像论坛上说的一样，express框架几乎不用怎么阅读文档就可以搞起了。最最常见的就是处理get和post请求了，处理过程中获取get和post数据相差很大，处理post请求还需要额外引入一些包。前端的ajax请求jQ已经帮我们封装好了，所以感知不到get和post的传值的不同。

前端对于post和get的不同处理

理解总结

get
追加在URL后通过序列化的字符串提交

post
协议规定 POST 提交的数据必须放在消息主体（entity-body）中，但协议并没有规定数据必须使用什么编码方式。实际上，开发者完全可以自己决定消息主体的格式，只要最后发送的 HTTP 请求满足上面的格式就可以。
主要有四种POST数据提交方式，前两种格式比较常用
1 application/x-www-form-urlencoded（jq，默认的表单提交格式）
通常写法: param1=111&param2=222
在XMLHttpRequest2中创建了FormData格式，方便数据表现形式为

2 application/json（由于json格式的流行，这种格式被广泛接受）
3 text/xml
4 multipart/form-data
